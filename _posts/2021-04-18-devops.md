운영관리의 전략차원에서 다양한 협업방식이 변경

- jira
    - 업무 일정 도구

    - 업무 처리

    - 소스 작성, 서로 소스 점검 시간 필요
    - 빌드 툴로 빌드할 수 있음

## AWS 와 Docker

### AWS
1. 장점
    - 탄력적인 웹 규모 컴퓨팅
    - 다양한 API 사용
        - 빠르게 추가 삭제 가능
    - 유연한 클라우드 호스팅
    - 통합된 서비스
    - 안정성
    - 보안
2. 단점
    - 베어 메탈 성능을 원할 때
    - 웹페이지가 몇개뿐일 때
    - 솔루션에 적합
    - 가격(잘 사용하지 않으면 비쌈)
    -> 작은서비스 가 아니면 좋음

### AWS의 종류
![dev_process](/image/lecutre/aws_kind.png)
1. Server
    1. EC2(Elastic Compute Cloud)
        - 안전
        - 크기 조정
        - 클라우드에서 제공하는 웹 서비스
        - 장점
            - 자동 확장 및 축소
            - 완전 제어
            - 다양한 커맨드 api 제공
            - S3, RDS 통합성
            - 안정성 보안
    2. Lambda
        - 프로비저닝 관리 필요 X
        - 사용한 컴퓨팅 시간만큼 비용 지불
        - 모든 유형의 app , backend service 별도의 관리없이 실행 가능
        - 높은 가용성
        - 장점
            - aws service 쉬운 확장
            - S3 , DynamoDB custom api 가능
