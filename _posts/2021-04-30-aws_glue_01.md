## [What is glue?](https://docs.aws.amazon.com/ko_kr/glue/latest/dg/what-is-glue.html)

![glue](/image\AWS\glue\HowItWorks-overview.png)
> You define jobs in AWS Glue to accomplish the work that's required to extract, transform, and load (ETL) data from a data source to a data target. You typically perform the following actions:
>> 데이터 소스에서 데이터 타겟으로 실행되는 ETL작업을 성취하기 위해서 AWS glue에서 job을 정의할 수 있습니다. 일반적으로 아래와 같은 형태로 진행된다.

- For data store sources, you define a crawler to populate your AWS Glue Data Catalog with metadata table definitions. You point your crawler at a data store, and the crawler creates table definitions in the Data Catalog. For streaming sources, you manually define Data Catalog tables and specify data stream properties.
> 좌측 상단에 보면, 데이터 저장소에 있는 데이터 소스가 있다.  AWS Glue의 Data catalog를 메타데이터 테이블 정의서로 채우기 위해 crawler를 사용한다. crawler를 데이터 스토어에 포인트하면 crawler는 data catalog에 테이블 정의서를 채운다.
스트리밍 데이터에 대해서는 수동으로 해줘야 함.

- AWS Glue가 Data Catalog가 ETL job을 정의하기 위해 필요한 다른 metadata 를 정의함.
- AWS Glue에 script를 추가하여 이용 가능
- 시간 기반의 스케쥴링 가능.
- AWS Glue는 Apache Spark 환경 기반으로 ETL 작업을 진행합니다.
